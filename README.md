# 안면마비 기반 뇌졸중 조기 인지 (Transfer Learning + Grad-CAM)

얼굴 이미지에서 **안면마비(정상 vs 마비)** 를 자동 판별하는 이미지 분류기입니다.
사전학습 CNN(ResNet50)을 **전이학습**하고, **Grad-CAM** 으로 모델이 얼굴의 어느 부위를 보고 판단하는지까지 검증했습니다.

> ⚠️ 이 모델은 **진단 도구가 아닙니다.** 의료적 판단을 대체하지 않으며, "병원에 가보라"고 권하는 비임상 스크리닝 보조 프로토타입입니다.

---

## 1. 왜 이 프로젝트인가 (Motivation)

- 뇌졸중은 **골든타임(약 4.5시간)** 안에 치료해야 후유증이 적지만, 절반 이상의 환자가 이 시간을 놓칩니다.
- 국제 표준 인지법 **FAST** 의 첫 글자 **F(Face)** 가 곧 **안면마비** — "미소 지을 때 한쪽 입꼬리가 처지는지"를 봅니다.
- 안면마비는 FAST 징후 중 **카메라만으로 비전문가도 관찰 가능한 거의 유일한 신호**입니다.
- 따라서 "얼굴 사진 → 마비 의심 여부 자동 판별"은 **일반인이 빠르게 위험을 인지하도록 돕는 스크리닝 보조**로서 의미가 있습니다.

---

## 2. 접근 방법 (Approach)

| 단계 | 내용 |
|---|---|
| 데이터 | 공개 데이터셋 사용 (마비: Kaggle facial-droop, 정상: human-faces) |
| 전처리 | **얼굴 크롭 + 정면 얼굴 필터**(OpenCV) — 배경 제거 + 포즈 통제 |
| 모델 | **ResNet50 전이학습** (ImageNet 사전학습, 분류 헤드 교체) |
| 평가 | 정확도 · 정밀도 · **재현율** · F1 · 혼동행렬 |
| 해석 | **Grad-CAM** — 모델이 입꼬리·눈가 등 의학적 부위를 보는지 검증 |

> 단순히 모델을 돌려 정확도를 내는 데 그치지 않고, **데이터 편향을 인지하고 통제한 뒤 모델의 판단 근거까지 검증**하는 데 초점을 뒀습니다.

---

## 3. 결과 (Results)

검증셋(validation) 기준:

| 지표 | 값 |
|---|---|
| 정확도 (Accuracy) | **약 96%** |
| 재현율 (Recall, 마비) | **약 0.93** |
| 혼동행렬 | 마비 169개 중 158개 검출(놓침 11) / 정상 192개 중 190개 정확 |

- 학습 곡선에서 train·val이 함께 수렴 → **과적합은 크지 않음.**
- **Grad-CAM 검증**: 모델이 마비 판단 시 **입·턱 부위**, 정상 판단 시 **얼굴 중앙(눈·이마)** 을 근거로 사용 → 의학적 소견(입꼬리 비대칭)과 모델 근거가 대체로 일치.

> 자세한 그래프(학습 곡선·혼동행렬·Grad-CAM 히트맵)는 노트북 출력에서 바로 확인할 수 있습니다.

---

## 4. 한계 (Limitations)

- **데이터 출처 편향(가장 중요)**: 마비 데이터(환자 영상 프레임)와 정상 데이터(웹 얼굴)는 출처가 달라, 모델이 얼굴이 아니라 화질·배경으로 구분할 위험이 있었습니다. 얼굴 크롭/정면 필터로 줄였고 Grad-CAM으로 검증했지만 **완전히 제거하진 못했습니다.** 즉 96% 정확도에는 실제 마비 식별 능력과 출처 차이가 함께 섞여 있을 수 있습니다.
- **데이터 규모/품질**: 공개 데이터라 규모가 작고 라벨 신뢰도가 낮습니다. 정식 연구라면 임상의가 라벨링한 YFP·MEEI 데이터가 필요합니다.
- **임계값 트레이드오프**: 재현율(놓치지 않기)을 높이면 오탐이 늘어납니다. 스크리닝 목적상 재현율 우선이 맞지만 비용이 있습니다.
- **진단이 아님**: 의료 판단을 대체하지 않습니다.

---

## 5. 다음 단계 (Future Work)

1. 임상 라벨 데이터(YFP/MEEI) 확보 후 재학습 → 신뢰도 향상
2. 마비/정상을 **같은 출처·같은 촬영조건**으로 맞춰 편향 제거
3. 재현율 중심 임계값 튜닝 + ROC 곡선 분석
4. 데이터 부족 보완을 위한 **GAN 기반 증강** → 증강 전/후 성능 비교

## 6. 실행 방법 (How to Run)

Google Colab 권장 (무료 GPU).

1. `facial_palsy_transfer_learning.ipynb` 를 Colab에서 열기 (런타임 → GPU)
2. Kaggle API 토큰을 Colab Secrets(🔑)에 `KAGGLE_TOKEN` 으로 등록
3. 위에서부터 순서대로 실행 (데이터 다운로드 → 정상 데이터 합치기 → 얼굴 크롭 → 학습 → 평가 → Grad-CAM)

**주요 라이브러리**: TensorFlow / Keras, OpenCV, scikit-learn, Matplotlib
