# stroke-facial-asymmetry-screening
안면 비대칭 기반 안면마비 의심 신호 탐지(뇌졸중 전조 스크리닝 보조) - 팀 프로젝트 정리

## 1) 프로젝트 개요
- 목표: 얼굴 이미지에서 안면 비대칭 특징을 기반으로 의심 신호를 탐지하고, 사용자가 의료기관 상담을 고려하도록 돕는 스크리닝 보조 프로토타입
- 주의: 본 프로젝트는 의료 진단이 아니며, 연구/학습 목적의 비임상 실험입니다.
- 성격: 2023년에 수행한 팀 프로젝트 산출물 기반 정리이며, 당시 데이터/외부 모델 파일 미포함으로 전체 재현은 제한될 수 있습니다(아래 참고).

## 2) 내가 맡은 역할(팀 기여)
- 학습/평가용 데이터 수집 및 정리
- 데이터 시각화 및 통계 분석(분포, 특성 비교 등)
- 실험 결과 정리 및 문서화(보고서 작성 기여)

## 3) 접근 방법(Pipeline)
1. 얼굴 검출 및 전처리(정렬/크롭 등)
2. Facial landmark 추출
3. 좌/우 비대칭 지표 계산(거리/각도/비율 기반)
4. 임계치 기반 판단 또는 분류 모델 기반 비교
5. 실패 케이스 분석 및 개선안 도출

## 4) 레포 구성(산출물)
- 노트북
  - notebooks/뇌졸중_통계.ipynb
  - notebooks/안면인식을_활용한_뇌졸중_의심환자_자가진단_서비스_ipynb의_사본.ipynb
- 결과보고서(PDF)
  - docs/프로젝트 결과보고서.pdf
- 결과 이미지
  - assets/results.png

## 5) 실행(재현) 관련 안내
본 레포는 코드보다 접근 방식/분석/결과 정리에 초점을 둔 케이스 스터디 성격이 강합니다.
특히 아래 항목이 레포에 포함되어 있지 않다면, 노트북 전체 실행은 환경 의존으로 제한될 수 있습니다.

- 얼굴 landmark predictor 파일(예: shape_predictor_68_face_landmarks.dat 등)
- 원본 이미지 데이터셋 및 라벨
- Colab/Drive 등 특정 경로에 의존하는 데이터 경로

그래서 본 레포는 다음 중 하나의 방식으로 확인하는 것을 권장합니다.
- 문서 중심 확인: docs 결과보고서 + assets 결과 요약 이미지 확인
- 노트북 확인: 파이프라인/지표 계산 로직 및 분석 흐름 확인(전체 실행이 아닌 코드/흐름 검토)

## 6) 결과 요약
- 실험 세부 결과 및 분석은 결과보고서에 정리되어 있습니다.

## 7) 한계(known issues)
- 촬영 각도(고개 기울임), 표정 변화, 조명, 가림(마스크/머리카락)에 민감하여 오탐 가능
- 데이터 규모/라벨 신뢰도 제한으로 일반화 성능을 보장하기 어려움
- 환경 의존(외부 파일/경로)로 인해 전체 재현에 제약이 발생할 수 있음

## 8) 개선 방향
- head pose(정면 여부) 추정 및 보정, 품질 필터링(blur/가림 감지) 적용
- 데이터 확장(각도/조명/표정 다양화) 및 라벨 기준 정교화
- landmark 추출을 설치가 쉬운 방식으로 대체(mediapipe 기반 데모로 재구성)하여 재현성 강화
